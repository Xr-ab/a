# CLAUDE.md

## 角色

AI应用开发学习者，用 Claude Code 做两件事：

1. 学习 Python/LangChain/RAG/Agent/大模型API 等技术栈
2. 独立开发个人AI项目（需求→代码→调试→落地）

## 思维原则

- 回到问题本质，不被惯例束缚；直接给真实判断，不谄媚
- 以学习为目的：方案要清晰易懂、可拆解复盘，避免黑箱实现

## 约束

- 新项目/新目录第一步建规范（CLAUDE.md + 结构约定），没有规范不动手
- 已有项目严格遵守其 CLAUDE.md；改规范先改文档再改实践
- 学习代码按模块化组织，每个功能点独立文件/文件夹
- 会话内已读取过的文件，后续复用一律走缓存，禁止重复加载

## 沟通

- 中文沟通，代码/命令/变量名用英文，关键注释用中文
- 结论先行再给理由；不铺垫背景
- 核心知识点（RAG/Agent/提示词工程）给出代码方案时补充简短原理说明

## 上下文管理

- 长会话中超出多轮后，对历史对话做摘要压缩：保留关键事实和决策结果，丢弃逐字对话细节和重复中间步骤
- 历史格式化输出结构化摘要，不完整保留所有原始消息
- 函数定义（名称、描述、参数 schema）首次传递完整定义，后续复用避免重复注入，节省 token
- 跨项目通用规则放在用户级 CLAUDE.md，项目专属规则放在项目级 CLAUDE.md，避免重复维护

## 红线（必须先问）

1. 删除文件/目录/git历史
2. 修改 .env/密钥/token/API Key/CI配置
3. 数据库schema变更（含本地向量库/SQLite）
4. git push / rebase / reset --hard / 强制推送
5. 安装全局依赖或修改系统配置
6. 公开发布/部署生产/发布npm包
7. 付费API高成本调用
8. 核心架构重大修改（换框架/重构目录）

## 工程纪律

- 改完主动验证，不只改不验
- 不注释报错不绕过，必须绕过时标注原因+修复建议
- 密钥/token/密码不进代码不进commit不进日志，用.env管理且加.gitignore
- 大改动前Plan Mode出方案，我确认后动手；完成后总结核心要点和学习收获
- 学习项目每个功能实现后补「复盘笔记」：实现逻辑、关键知识点、可优化方向
